# 部署教程

本文档说明如何把“家庭共享相册”部署到 Railway，并正确保留图片、缩略图与照片元数据。

## 推荐方案：Railway

这个项目最适合 Railway，因为它支持 Node.js 服务和持久化 Volume，适合保存：
- `uploads/` 中的原图
- `thumbnails/` 或 `uploads/thumbnails/` 中的缩略图
- `photo-data.json` 中的评论、表情、描述、标签、收藏、排序、分组数据

## 部署前准备

1. 拥有 GitHub 账号
2. 拥有 Railway 账号
3. 已把项目代码推送到 GitHub 仓库

## 第一步：推送代码到 GitHub

```bash
git init
git add .
git commit -m "init family album"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

## 第二步：在 Railway 创建项目

1. 打开 `https://railway.app`
2. 选择 `Deploy from GitHub repo`
3. 连接 GitHub
4. 选择这个项目仓库
5. Railway 会自动识别为 Node.js 项目并开始部署

## 第三步：添加持久化 Volume

如果不加 Volume，图片、缩略图和 `photo-data.json` 可能会在重启后丢失。

操作方式：
1. 在 Railway 项目中点击 `Add Service`
2. 选择 `Volume`
3. Mount Path 填写：`/data`

## 第四步：配置环境变量

在 Railway 的 `Variables` 中设置：

```text
RAILWAY_VOLUME_MOUNT_PATH=/data
DELETE_PASSWORD=你的删除密码
PORT=3000
```

说明：
- `RAILWAY_VOLUME_MOUNT_PATH=/data` 用于把图片、缩略图和 JSON 数据写入持久化目录
- `DELETE_PASSWORD` 是删除图片时使用的密码
- `PORT` 通常 Railway 会自动处理，不填也可以

## 第五步：访问站点

部署完成后：
1. 打开项目设置
2. 在 `Domains` 中生成 Railway 域名
3. 访问生成的 `*.railway.app` 地址

## 项目当前会持久化什么

部署后以下内容会保存在 Volume 中：
- 上传图片文件
- 缩略图文件
- 点赞数据
- 评论数据
- 表情回应数据
- 照片描述 `caption`
- 收藏状态 `favorited`
- 标签 `tags`
- 手动拖拽排序 `order`
- 分组信息 `groupName`

## 更新部署

后续修改后重新推送即可自动部署：

```bash
git add index.html style.css js server server.js README.md CLAUDE.md CODEX.md 部署教程.md
git commit -m "update album"
git push origin main
```

## 本地局域网访问

如果你不想部署云端，也可以本地运行：

```bash
npm install
npm start
```

然后让家人访问：

```text
http://你的局域网IP:3000
```

Windows 查看局域网 IP：

```bash
ipconfig
```

## 常见问题

### 1. 为什么我部署后图片会丢失？
通常是没有挂载 Volume，或者没有配置 `RAILWAY_VOLUME_MOUNT_PATH`。

### 2. 为什么排序刷新后没保留？
拖拽排序会写入 `photo-data.json` 的 `order` 字段，所以必须保证 JSON 文件也在持久化目录内。

### 3. 为什么收藏、标签、描述、分组刷新后没保留？
这些信息都写在 `photo-data.json` 中，所以同样需要保证该文件在 Volume 中持久化。

### 4. `photo-data.json` 不存在正常吗？
正常。项目第一次写入评论、标签、排序、收藏等数据时会自动生成。

### 5. 删除密码忘了怎么办？
重新在 Railway 的环境变量里修改 `DELETE_PASSWORD` 即可。

## 建议

长期使用建议：
- 使用 Railway + Volume
- 定期备份 `uploads/`、缩略图目录和 `photo-data.json`
- 如果后面照片量变大，再考虑迁移到 SQLite 或对象存储
