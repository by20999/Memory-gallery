# `memory/` 项目长期记忆

本目录用于保存项目日志、决策记录和长期维护备注。

## 当前状态

历史零散记忆文档已合并到根目录和 `docs/` 文档体系中。本目录保留作为后续记录区。

## 建议文件

| 文件 | 作用 |
| --- | --- |
| `decision-log.md` | 记录重要技术决策。 |
| `release-notes.md` | 记录版本或阶段性变更。 |
| `incident-log.md` | 记录线上故障和修复。 |

## 使用规则

- 项目正式说明放在 `README.md` 和 `docs/`。
- 长期背景、决策理由和运维事件可以放在这里。
- 影响项目定位、架构边界、存储策略或维护习惯的重要取舍，记录到 `decision-log.md`。
- 阶段性功能和体验更新，记录到 `release-notes.md`；普通建设流水优先记录到 `../docs/project-log.md`。
- 不要保存密码、Token 或私密照片信息。
